# Представлена работа с джойстиком KY-023 + двойным светодиодом KY-011.  

KY-023 — это двухкоординатный модуль-джойстик, который используется в робототехнике и проектах на Arduino для управления движением.  
По своему принципу работы он полностью аналогичен стикам на геймпадах от игровых приставок (например, PlayStation или Xbox).  

## Как он устроен  
В основе модуля лежат три главных компонента:  

* Два потенциометра (переменных резистора): один отслеживает движения по оси X (влево-вправо), другой — по оси Y (вверх-вниз). При отклонении ручки меняется сопротивление.  
* Тактовая кнопка: срабатывает при вертикальном нажатии на саму ручку джойстика.  
* Пружинный механизм: автоматически возвращает ручку в центральное положение, когда вы её отпускаете.  

## Назначение выводов (пинов)  
Модуль имеет 5 контактов для подключения:  

* GND: заземление (минус питания).  
* +5V: питание модуля (+5 Вольт от платы управления).  
* VRx: аналоговый выход оси X. Выдает напряжение от 0 до 5 В в зависимости от положения.  
* VRy: аналоговый выход оси Y. Работает так же, как и ось X.  
* SW: цифровой выход встроенной кнопки. При нажатии замыкается на GND.  

## Принцип работы  

* В режиме покоя (центр): на выходах VRx и VRy выдается примерно половина напряжения питания (около 2.5 В). Микроконтроллер считывает это значение как середину шкалы (например, 512 при 10-битном АЦП Arduino).  
* При движении: напряжение на выходах VRx и VRy плавно меняется от 0 до 5 В в зависимости от направления и угла наклона ручки.  

## Где применяется  

* Управление колесными роботами и гусеничными платформами.  
* Управление сервоприводами (например, манипулятором или наклоном камеры).  
* Навигация по меню на ЖК-дисплеях в различных приборах.  
* Создание самодельных игровых контроллеров.  

Для подключения джойстика KY-023 к плате Nucleo-STM32F411RE вам понадобятся два аналоговых входа (ADC) для осей X и Y, а также один цифровой вход для кнопки.  
Так как плата Nucleo работает с логическим уровнем 3.3 В, модуль необходимо питать от источника 3.3 В, чтобы не повредить аналоговые входы микроконтроллера.  

## Схема подключения  
Используйте контакты разъема Arduino Uno V3 (размещены по краям платы Nucleo) для удобства:  

| Вывод KY-023 | Вывод Nucleo-F411RE | 			 Описание  	     |  
|--------------|---------------------|-----------------------|  
|    GND 		   |       GND           | Общая земля			     |  
|    +5V       |       3.3V 				 | Питание (строго 3.3 В)|  
|    VRx       |       A0 (PA0)			 | Аналоговый вход оси X |  
|    VRy       |       A1 (PA1)			 | Аналоговый вход оси Y |  
|    SW        |       D2 (PA10)		 | Цифровой вход кнопки	 |

##  Настройка в STM32CubeMX  
Если вы пишете код через STM32CubeIDE, настройте периферию следующим образом:  

   1. Аналоговые оси (X и Y):  
   * Включите ADC1.  
      * Активируйте каналы IN0 (пин PA0) и IN1 (пин PA1) в режиме Internal Trigger / Regular Conversion.  
      * В настройках ADC (Configuration) включите Scan Conversion Mode (в положение Enabled) и установите Number of Conversion равным 2.  
      * Настройте порядок опроса (Rank): Rank 1 — Channel 0 (ось X), Rank 2 — Channel 1 (ось Y).  
      * Для стабильности рекомендуется включить Continuous Conversion Mode или использовать DMA для чтения этих двух каналов.  
   2. Кнопка (SW):  
   * Нажмите на пин PA10 (D2) и выберите режим GPIO_Input.  
      * В настройках GPIO для этого пина обязательно выберите Pull-up (внутренний подтягивающий резистор). Так как кнопка замыкается на землю, в отжатом состоянии на пине должно быть 3.3 В.

На левой стороне платы Nucleo-F411RE (на разъеме CN6, который относится к секции Power) есть отдельные силовые выводы питания.  
## Расположение контактов (сверху вниз):  

* 3V3 — выход стабилизатора платы (3.3 Вольта).  
* 5V — выход 5 Вольт (питается от USB или внешнего источника VIN).  
* GND — земля (минус питания, там доступно два контакта подряд).  

## Важное предупреждение для KY-023:  
Хотя на плате есть вывод 5V, подключать пин +5V джойстика к нему нельзя.  
Если вы подадите на джойстик 5 Вольт, то при максимальном отклонении ручки джойстик выдаст на аналоговые выходы (VRx, VRy) тоже 5 Вольт.  
Входы микроконтроллера STM32F411RE рассчитаны строго на напряжение до 3.3 В. Подача 5 Вольт на аналоговый пин АЦП может сожчь порт микроконтроллера.  
Правильное решение: Подключайте пин +5V джойстика именно к разъему 3V3 на Nucleo.  
Модуль отлично работает от 3.3 В, а значения АЦП будут безопасно меняться от 0 до 3.3 В (от 0 до 100% в коде).

Пошаговая проверка схемы, чтобы всё заработало с первого раза и плата осталась в безопасности.  
Для проверки сверьтесь со списком ниже. Провод от каждого пина джойстика должен идти строго на свой контакт платы Nucleo.  

## Чек-лист проверки контактов  

   1. Питание (Самое важное!):
      * Пин +5V джойстика подключен к контакту 3V3 в разъеме Power на Nucleo. (Проверьте еще раз: провод НЕ должен быть вставлен в разъем 5V).  
      * Пин GND джойстика подключен к любому из контактов GND в разъеме Power на Nucleo.  
   2. Аналоговые оси:  
      * Пин VRx джойстика подключен к аналоговому контакту A0 (разъем CN8, сразу под блокм Power).   
      * Пин VRy джойстика подключен к аналоговому контакту A1 (разъем CN8, сразу под пином A0).  
   3. Кнопка:  
     * Пин SW джойстика подключен к цифровому контакту D2 (разъем CN9, правая сторона платы).

## Физический тест схемы (без прошивки)  
Если у вас есть мультиметр, можно провести быстрый тест безопасности ПЕРЕД включением платы в компьютер:  

   1. Переключите мультиметр в режим измерения сопротивления (Ом).  
   2. Измерьте сопротивление между пинами +5V и GND прямо на джойстике.  
   3. Результат: оно не должно быть нулевым (короткого замыкания быть не должно). Обычно там будет около 2.5–5 кОм, в моём случае было 3.9 кОм.  

Если всё подключено именно так, схема полностью безопасна и готова к подаче питания.  

Можно увидеть цифры перемещения джойстика на экране своего компьютера через монитор последовательного порта (Serial Terminal).  
Микроконтроллер будет считывать положение джойстика и отправлять текстовые данные через встроенный программатор ST-LINK по USB-кабелю прямо в ПК.

Чтобы настроить периферию в STM32CubeIDE и запустить непрерывное чтение двух каналов АЦП без зависания процессора, лучше всего использовать связку ADC + DMA.  
Если опрашивать два канала обычной функцией HAL_ADC_PollForConversion, они будут перезаписывать значения друг друга, так как регистр данных у АЦП всего один.  
DMA же автоматически разнесет значения осей X и Y в разные ячейки памяти.  
Пошаговая инструкция для настройки графического конфигуратора (.ioc файла) и готовый рабочий код.  
------------------------------  
##  Настройка в графическом редакторе CubeMX (.ioc)  

   1. Включение каналов:  
	   * Слева в меню откройте Аналоговые (Analog) -> ADC1.  
      * Поставьте галочки напротив IN0 и IN1. (Пины PA0 и PA1 на схеме микроконтроллера автоматически окрасятся в зеленый цвет).  
   2. Настройка параметров АЦП (Вкладка Configuration -> Parameter Settings):  
	   * Clock Prescaler: PCLK2 divided by 4 (или 2, для джойстика скорость не критична).  
      * Resolution: 12-bit (значения от 0 до 100%).  
      * Scan Conversion Mode: Поставьте Enabled (активируется автоматически при выборе нескольких каналов).  
      * Continuous Conversion Mode: Поставьте Enabled (АЦП будет циклически опрашивать оси одну за другой без остановки).  
      * End Of Conversion Selection: End of single conversion.  
      * Number Of Conversion: Измените 1 на 2.  
   3. Настройка порядка опроса (Раздел Rank):  
	   * Раскройте появившийся список Rank 1: выберите Channel 0 (это наша ось X).  
      * Раскройте список Rank 2: выберите Channel 1 (это наша ось Y).  
      * Совет: В параметрах Sampling Time для обоих рангов поставьте значение повыше (например, 56 Cycles или 84 Cycles), это сгладит шумы джойстика.  
   4. Включение DMA (Вкладка Configuration -> DMA Settings):  
	   * Нажмите кнопку Add.  
      * В появившейся строке выберите ADC1.  
      * В нижней части настроек DMA (DMA Request Settings) проверьте:  
      * Mode: Переключите с Normal на Circular (круговой режим, чтобы буфер обновлялся бесконечно).  
         * Increment Address: Поставьте галочку только на Memory (АЦП пишет в одну точку, а DMA сдвигает адрес в массиве памяти).  
         * Data Width: Для Peripheral и Memory выберите Half Word (16 бит, так как результат АЦП 12-битный).  
   5. Настройка кнопки и UART (Для вывода данных):  
	   * Кликните на пин PA10, выберите GPIO_Input. В меню Connectivity -> GPIO для PA10 в строке Pull-up/Pull-down выберите Pull-up.  
      * Зайдите в Connectivity -> USART2, выберите режим Asynchronous, скорость (Baud Rate) проверьте — 115200.  
   
Нажмите Ctrl + S (или Project -> Generate Code), чтобы CubeIDE сгенерировал проект.

## Шаг 1: Программа на ПК для просмотра данных  
В зависимости от того, в чем вы пишете код, используйте один из вариантов:

   1. STM32CubeIDE: Внутри среды есть встроенная вкладка Console (нужно настроить Command Shell Console) или плагин STM32CubeMonitor.
   2. Можно использовать сторонние утилиты, например, Putty — классическая программа, выберите тип соединения Serial.  
   
## Шаг 2: Настройка подключения в терминале  
Когда вы запустите программу-терминал, вам нужно указать параметры подключения:  

* COM-порт: Выберите порт вашей платы (в Диспетчере устройств Windows он обычно называется STMicroelectronics ST-LINK Virtual COM Port).  
* Скорость (Baud Rate): 115200 (стандартное значение для большинства проектов STM32).  
* Data bits: 8  
* Stop bits: 1  
* Parity: None  

## Что вы увидите на экране:  
После прошивки платы и открытия терминала на экране побегут строки:  

* В центре: X: 0% | Y: 0% | Button: 0 (никакого дребезга)  
* До упора влево: X: 0% | Y: 100% | Button: 0  
* До упора вверх: X: 100% | Y: 100% | Button: 0  
* При нажатии на стик: X: 100% | Y: 100% | Button: 1

Для более наглядного показа работы джойстика, подсоединим KY-011 TWO-COLOR.  
Нажатие кнопки джойстика выведено на зеленый светодиод LD2, что на плате.  
Координаты X и Y джойстика подсвечиваются через KY-011.  

Модуль KY-011 (Two-color LED) представляет собой двухцветный светодиод (обычно красный и зеленый), совмещенный на одной плате.  
В отличие от предыдущего модуля KY-034, у KY-011 нет встроенной микросхемы — это просто два раздельных кристалла светодиода с общим катодом (минусом).  
Настроим схему так, чтобы при наклоне джойстика по оси X плавно разгорался один цвет (например, красный), а по оси Y — другой цвет (например, зеленый).  
В промежуточных положениях цвета будут смешиваться, выдавая желтые и оранжевые оттенки.

## Схема подключения KY-011 к Nucleo-F411RE  
Нам нужны два канала ШИМ, поэтому подключаем таймер TIM3 на пины PC6 и PC7.  

| Вывод KY-011 	|   Вывод Nucleo-F411RE	| 				Описание 					|  
|---------------|-----------------------|-------------------------------------------|  
|   R (Red) 	| 		PC6 (D3) 		| Управление красным цветом (Канал ШИМ 1) 	|  
|   G (Green) 	| 		PC7 (D9)		| Управление зеленым цветом (Канал ШИМ 2) 	|  
|   - (Minus) 	| 		GND 			| Общая земля (Общий катод) 				|  

Примечание: 
	* На модуле KY-011 обычно уже распаяны токоограничивающие резисторы, поэтому его можно подключать к пинам Nucleo напрямую.  
	* Контакты R (Red) и G (Green) обозначены на плате как S.
------------------------------  
## Настройка в STM32CubeMX (.ioc)  
Вам нужно вернуть настройки таймера TIM3, которые мы использовали для обычных светодиодов:  

   1. Откройте Timers -> TIM3.  
   2. Установите Clock Source в положение Internal Clock.  
   3. Для Channel 1 выберите PWM Generation CH1 (пин PC6 автоматически станет ШИМ-выходом).  
   4. Для Channel 2 выберите PWM Generation CH2 (пин PC7 также станет ШИМ-выходом).  
   5. В параметрах конфигурации (Parameter Settings) верните настройки частоты:  
   	* Prescaler (PSC): 95  
	* Counter Period (ARR): 999  
   6. Сохраните проект (Ctrl + S), чтобы CubeIDE перегенерировал код создал структуру htim3.

## Как теперь ведет себя модуль KY-011:  

* В центре: Светодиод полностью погашен.  
* Движение влево/вправо (ось X): Плавно разгорается красный цвет.  
* Движение вверх/вниз (ось Y): Плавно разгорается зеленый цвет.  
* Движение по диагонали: Оба кристалла светятся одновременно, формируя красивый желтый или оранжевый цвет в зависимости от угла наклона.

   В общем на пины PC6 (D3) и PC7 (D9) у меня подавалось постоянно напряжение порядка 0.35–0.4 В и оно не менялось при изменении положений джойстика.  
   Пришлось принудительно в коде перенастроить пин PC7 напрямую через регистры.  
   
```sh
/* USER CODE BEGIN 1 */
// 1. Включаем тактирование Порта C и Таймера 3 напрямую в регистрах
__HAL_RCC_GPIOC_CLK_ENABLE();
__HAL_RCC_TIM3_CLK_ENABLE();

// 2. Переводим пин PC7 в режим альтернативной функции (ШИМ таймера)
GPIOC->MODER &= ~(3 << (7 * 2)); // Очищаем старый режим
GPIOC->MODER |= (2 << (7 * 2)); // Устанавливаем режим 10 (Alternate function mode)

// 3. Указываем, что альтернативная функция для PC7 — это именно TIM3 (AF02)
GPIOC->AFR[0] &= ~(0xF << (7 * 4)); // Очищаем регистр альтернативных функций для пина 7
GPIOC->AFR[0] |= (2 << (7 * 4));   // Записываем код AF2 (Таймер TIM3)
/* USER CODE END 1 */
```
